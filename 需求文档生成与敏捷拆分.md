## Agent F 亚马逊智能客服 Agent (MVP v1.0) 需求文档

**项目名称**: Agent F 亚马逊智能客服 Agent (MVP v1.0) **文档版本**: v1.0 **编制日期**: 2026-04-03 

---

### 一、 产品概述

- **产品目标**: 为跨境卖家打造的一款智能客服工具，通过集成大模型与平台 API，提供翻译、意图识别及自动化草稿生成服务 。
- **交付形态**: 基于响应式设计的桌面端网页应用（优先适配 PC 端） 。
- **核心策略**: MVP 阶段坚决放弃全自动发信，强制实行“AI 生成草稿，人工点击确认发送”的人机协同模式，以降低退店业务风险 。

---

### 二、 技术架构与底座

- **后端开发框架**: 采用 Java / Spring Boot 3.x，以保障多租户环境下的高并发与高稳定性 。
- **AI 编排底座**: 使用 LangChain4j 替代低代码工作流，以实现代码级别的高精度控制 。
- **大模型服务**: 采用阿里云通义千问大模型 (Qwen-Plus) 及 text-embedding-v2 模型，物理隔离数据以规避出境合规风险 。
- **数据库架构**: 统一采用 PostgreSQL 处理关系型业务数据，并结合 pgvector 插件实现 RAG 向量检索 。

---

### 三、 核心功能需求

#### 3.1 亚马逊平台对接 (SP-API)

- **授权与店铺管理**: 支持卖家通过 OAuth 完成店铺授权，并提供清晰的数据使用合规承诺页 。
- **消息同步**: 基于 Amazon SQS 与 Notifications API 的事件驱动架构，后端需异步监听并拉取买家消息 。
- **订单信息查询**: 支持通过买家 ID 或订单号调用端点查询并返回所需订单字段 。
- **回复发送**: 支持获取订单后，调用指定 API 接口向买家发送合规消息 。

#### 3.2 AI 辅助与处理

- **消息预处理**: 新消息到达时异步触发 LLM 任务，实现翻译、摘要提取与意图分类 。
- **智能回复生成**: 结合上下文或相关知识为卖家自动生成回复草稿 。
- **合规校验**: 提供合规词库扫描及高亮警告，作为辅助审查手段 。
- **人工兜底优化**: 提供人工修正草稿机制，并需记录低置信度样本用于未来模型微调 。

#### 3.3 RAG 知识库管理

- **文档入库**: 支持卖家上传专属产品 PDF 或文档 。
- **解析机制**: 采用 pdfplumber 等工具进行解析，系统需为图文混合的复杂文档预留 OCR 版面分析通道 。
- **检索赋能**: 业务侧可检索入库的产品数据，用于辅助生成更精确的客服回复 。

#### 3.4 前端工作台交互

- **界面模块**: 需提供工作台消息列表、详情页、订单侧边栏展示及编辑发送框交互 。
- **实时通知**: 通过前端轮询结合 WebSocket 双通道，确保新消息桌面通知延迟小于 5 秒 。

#### 3.5 用户分级与权限管理

- **基础版 (Starter)**: 提供每月 500 条消息额度，功能仅支持 AI 意图识别与翻译草稿，限制 RAG 知识库功能 。
- **专业版 (Pro)**: 提供每月 2,000 条消息额度，支持上传专属产品 PDF 构建知识库，并开放多客服子账号功能 。

---

### 四、 非功能性需求

#### 4.1 安全与合规要求

- **数据隔离**: 必须按 `seller_id` 进行分表或分 schema 管理，保障多租户数据独立安全 。
- **数据销毁 (Data Retention Policy)**: 所有包含买家姓名、邮箱、地址等 PII（个人身份信息）的数据，必须在订单完结或消息处理完成的 30 天内执行硬删除销毁机制 。
- **敏感信息加密**: Token 等敏感密钥必须使用 AES-256 标准加密存储 。
- **模型隐私**: 接口调用需明确禁止大模型将交互数据用于模型训练 。

#### 4.2 性能与并发要求

- **接口限流控制**: 针对亚马逊默认的 1 次/秒/店铺 API 限制，系统必须实现基于 Redis 的令牌桶限流与指数退避重试机制 。
- **并发队列**: 需隔离不同店铺的异步处理队列，避免单点阻塞全盘系统导致崩溃 。
